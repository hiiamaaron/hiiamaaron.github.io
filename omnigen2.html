Multi-Modal Pipeline & Prompt Engineering

Product images are generated using a multi-step process:

  • Translate Korean prompts to English using GPT-4o-mini.
  • Refine prompts and extract key keywords using GPT-4o-mini.
  • Web search for relevant content and incorporate into prompt improvements.
  • Generate negative prompts to improve image quality.
  • Feed refined prompts to image generation model (OmniGen2 / Stable Diffusion).
  • Analyze generated image with OmniGen2 vision module and translate summary to Korean.
  • Generate final advertisement copy using GPT-4o-mini.

Back-End | FastAPI + SQLModel

  • Removed previous cache modules (cache.py and router cache).
  • Defined schemas, CRUD, and routers for communication with SQLModel database and generative models (e.g., Stable Diffusion, GPT-4o-mini).
  • Generated images are saved to server disk (backend/static/generated_images/) with SHA hash-based filenames.
  • Password encryption, login verification, and session authentication implemented.

SQL Database | SQLModel

  • Session: Unique session IDs for users stored in database.
  • User: Stores username, encrypted password, email.
  • Advertisement: Stores user ID and prompt input.
  • AdvertisementImagePreservation: Stores generated background images.
  • AdvertisementCopy: Stores generated advertisement text.

System Limitations & Performance

OmniGen2 required ~38 GB VRAM just for initialization, exceeding the available 24 GB on GCP L4 NVIDIA GPU. Therefore, Stable Diffusion + GPT were used for deployment.
